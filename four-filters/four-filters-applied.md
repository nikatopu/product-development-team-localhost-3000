## Team Discussion Notes

We compared API communication, design handoff, freelance estimation, and admin communication.

Admin communication was dropped due to dependency on university systems and lack of access to decision-makers. Freelance estimation was considered too broad and hard to scope into a clear MVP. Design handoff was strong, but existing tools already partially solve it and the problem depends on designer behavior.

API communication stood out as the best option. It is a daily problem we all experience, has a clear workaround (manual docs and messaging), and existing tools are not well integrated into real workflows. Most importantly, we are the target users and can start validation immediately without external dependencies.

This made API communication the most feasible and practical choice for the project.elance estimation are strong alternatives, API communication provides the best balance of real pain, clear solution space, and feasibility within the project constraints.

## Decision

Primary Problem:
**Improving communication of API contracts between backend and frontend developers (12/12)**

Secondary Candidates:

- Design handoff issues (11/12)

# Experiment Results — Driftless

## Experiment Overview

Driftless is a developer-focused software tool that analyzes ASP.NET / .NET backend repositories and automatically generates API documentation directly from source code. The product is designed to solve the common problem of outdated, missing, or manually maintained API documentation, which often creates delays and communication issues between backend and frontend developers.

The purpose of this experiment was to validate whether developers and computer science students recognize API documentation as a real problem and whether they would consider using an automated documentation generator like Driftless in real development environments.

---

## Hypothesis

We believe showing developers a working Driftless MVP that automatically generates API documentation from ASP.NET repositories will cause increased interest and willingness to adopt the product among backend, frontend, and full-stack developers.

We will know this is true when at least 60% of tested users say they would actively use Driftless in real projects or recommend it to teammates.

---

## Method

**Conducted:** May 10–14, 2026  
**Format:** In-person and remote product concept demonstrations with structured feedback collection  
**Conducted by:** Giorgi Tkebuchava, Toma Danelia

### Process:

1. Participants were introduced to the problem:
   - API documentation is often incomplete or outdated.
   - Frontend teams frequently depend on backend clarification.
   - Manual documentation maintenance slows down projects.

2. Driftless MVP concept was demonstrated:
   - User inputs GitHub repository URL
   - Driftless scans ASP.NET backend code
   - Routes/endpoints are automatically extracted
   - Documentation is generated

3. Participants answered validation questions:
   - Have you experienced API documentation problems?
   - Would automated API documentation save development time?
   - Would you personally try Driftless?
   - What improvements or concerns would you have?

---

## Sample

- Total participants: 10
- User segment:
  - Computer science students
  - Junior software developers
  - Backend/frontend project collaborators
- Validation type:
  - Product demo
  - Direct questioning
  - User feedback analysis

---

## Results

| Metric                                              | Result |
| --------------------------------------------------- | ------ |
| Participants tested                                 | 10     |
| Participants who experienced API documentation pain | 8      |
| Participants who found the product useful           | 8      |
| Participants willing to actively try Driftless      | 7      |
| Participants requesting broader framework support   | 6      |

---

## Success Threshold

### Target:

At least 60% of participants must express willingness to use or recommend Driftless.

### Actual:

7 / 10 = 70%

### Outcome:

The experiment exceeded the success threshold.

---

## Qualitative Findings

### Main Positive Feedback:

- Developers quickly understood the problem because API documentation issues are common in collaborative projects.
- The strongest value proposition was identified for teams where frontend and backend development are separated.
- Participants appreciated that Driftless reduces manual documentation workload by reading real code directly.
- Several participants viewed Driftless as potentially useful for student projects, startups, and internal team environments.

### Main Concerns:

- Current MVP is highly focused on ASP.NET / .NET repositories, limiting broader market reach.
- Participants consistently requested future support for frameworks such as FastAPI, Express.js, Django, and Spring Boot.
- Documentation accuracy must remain high, as unreliable generated documentation would reduce trust.
- Some users viewed Driftless more as a niche developer productivity tool unless expanded into multi-language support.

---

## Decision: Persevere (With Planned Expansion)

### Final Decision:

**Persevere with Driftless as a focused .NET MVP while strategically planning future framework expansion.**

### Reasoning:

The primary validation metric was user willingness to adopt Driftless.

- Success threshold: 60%
- Actual result: 70%
- Willing participants: 7 out of 10

Because Driftless exceeded its target validation threshold, the product demonstrates sufficient market interest to justify continued development.

### Strategic Conclusion:

- Immediate focus remains on strengthening .NET/ASP.NET endpoint extraction
- Product-market fit appears viable for developer teams within the .NET ecosystem
- Long-term growth will likely require expansion beyond .NET to reach larger developer markets

---

## Pivot Analysis

### Why Full Pivot Is Not Recommended:

- Core problem is validated
- Initial market demand exists
- MVP has clear technical feasibility
- Narrow niche can still support early adoption

### Why Roadmap Expansion Is Necessary:

- Market size is currently constrained
- Cross-framework compatibility would significantly increase growth potential
- Broader language support improves long-term SaaS scalability

## Final Conclusion

The validation sprint confirms that Driftless solves a legitimate developer pain point and has measurable early-stage market demand.

While the current product is niche due to framework specialization, the experiment supports continued development rather than pivoting away from the concept.

Driftless should proceed as a focused ASP.NET documentation generator MVP while strategically expanding toward a broader multi-framework developer platform.
